# US-08: Оплата поездки

## 📜 Формулировка требования

Как пассажир, я хочу оплатить поездку удобным способом, чтобы завершить заказ.

## 👥 Роли пользователей

- Все пассажиры с завершенной поездкой
- Студенты (Apple Pay/Google Pay/СБП)
- Бизнес-клиенты (корпоративный счет)
- Пенсионеры (наличные)

## 🏷 Атрибуты

| Параметр | Значение |
|----------|----------|
| Источник | Use Case UC-17, KANO F8,F9 |
| Приоритет | Must Have |
| Стабильность | Стабильно |
| Сложность | Высокая |

## 📋 Детализация по персонам

### Студент Игорь (20 лет)
**User Story:** Как студент, я хочу оплатить через Apple Pay / Google Pay / СБП, чтобы не вводить данные карты.

**Критерии приемки:**
- [ ] Доступны все популярные способы бесконтактной оплаты
- [ ] Оплата происходит в один клик
- [ ] Подтверждение оплаты приходит мгновенно
- [ ] Возможность сохранить несколько карт

### Бизнес-аналитик Игорь (38 лет)
**User Story:** Как сотрудник, я хочу оплата списывалась с корпоративного счета автоматически, чтобы не использовать личные деньги.

**Критерии приемки:**
- [ ] Корпоративный счет доступен как способ оплаты
- [ ] Система проверяет лимит и активность аккаунта
- [ ] Чек автоматически отправляется на корпоративную почту
- [ ] Детализация поездки включается в отчет

### Пенсионер Игорь Александрович (67 лет)
**User Story:** Как пенсионер, я хочу оплатить наличными водителю, чтобы не привязывать карту.

**Критерии приемки:**
- [ ] Способ оплаты "Наличные" доступен всегда
- [ ] Сумма к оплате отображается крупно
- [ ] Водитель подтверждает получение наличных
- [ ] Статус заказа меняется на "Оплачено"

## 📊 Метрики успеха

- Процент успешных платежей ≥ 99%
- Время обработки платежа ≤ 5 секунд
- Процент пользователей, использующих наличные: 15% (целевой сегмент 60+)
