# US-05: Подтверждение заказа и применение промокодов

## 📜 Формулировка требования

Как пассажир, я хочу подтвердить заказ и применить промокод (если есть), чтобы создать заказ и начать поиск водителя.

## 👥 Роли пользователей

- Все аутентифицированные пассажиры
- Студенты (активно используют промокоды)
- Бизнес-клиенты (заказывают на время)
- Пенсионеры (нужна большая кнопка подтверждения)

## 🏷 Атрибуты

| Параметр | Значение |
|----------|----------|
| Источник | Use Case UC-04 (шаги 6-9), KANO F12 |
| Приоритет | Must Have |
| Стабильность | Стабильно |
| Сложность | Средняя |

## 📋 Детализация по персонам

### Студент Игорь (20 лет)
**User Story:** Как студент, я хочу применить промокод перед подтверждением заказа, чтобы получить скидку и сэкономить.

**Критерии приемки:**
- [ ] Поле для ввода промокода доступно перед подтверждением
- [ ] При вводе валидного промокода цена пересчитывается
- [ ] Отображается размер скидки
- [ ] Можно удалить промокод и вернуть исходную цену

### Бизнес-аналитик Игорь (38 лет)
**User Story:** Как сотрудник, я хочу запланировать поездку на конкретное время (например, 9:15), чтобы быть уверенным в подаче.

**Критерии приемки:**
- [ ] Доступен переключатель "Заказать сейчас / Заказать на время"
- [ ] При выборе "на время" открывается календарь и выбор времени
- [ ] Можно выбрать дату и время с шагом 5 минут
- [ ] Система создает отложенный заказ с приоритетным поиском

### Пенсионер Игорь Александрович (67 лет)
**User Story:** Как пенсионер, я хочу видеть кнопку «Вызвать такси» очень большой и контрастной, чтобы не ошибиться.

**Критерии приемки:**
- [ ] Кнопка "Заказать такси" занимает минимум 48x48 dp
- [ ] Контрастность кнопки соответствует WCAG 2.1 AA
- [ ] Текст кнопки четкий и крупный (минимум 18sp)
- [ ] При нажатии требуется подтверждение (защита от случайного)

## 📊 Метрики успеха

- Конверсия из подтверждения в созданный заказ ≥ 90%
- Процент успешного применения промокодов ≥ 95%
- Время от нажатия "Заказать" до создания заказа ≤ 3 секунды
