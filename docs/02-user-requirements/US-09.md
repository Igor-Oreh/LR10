# US-09: Получение чеков и отчетов

## 📜 Формулировка требования

Как пассажир, я хочу получить чек и отчет о поездке, чтобы использовать его для бухгалтерии или личного учета.

## 👥 Роли пользователей

- Все пассажиры с завершенными поездками
- Студенты (кэшбэк бонусы)
- Бизнес-клиенты (автоматическая выгрузка)

## 🏷 Атрибуты

| Параметр | Значение |
|----------|----------|
| Источник | Use Case UC-17, KANO F6 |
| Приоритет | Should Have (для B2B - Must Have) |
| Стабильность | Стабильно |
| Сложность | Средняя |

## 📋 Детализация по персонам

### Студент Игорь (20 лет)
**User Story:** Как студент, я хочу видеть кэшбэк бонусы сразу после поездки, чтобы отслеживать накопления.

**Критерии приемки:**
- [ ] После оплаты отображается сумма кэшбэка
- [ ] Бонусы зачисляются на счет мгновенно
- [ ] Доступна история начислений
- [ ] Можно потратить бонусы на следующую поездку

### Бизнес-аналитик Игорь (38 лет)
**User Story:** Как бухгалтер/аналитик, я хочу автоматически получать акты и чеки в Excel/PDF на почту, чтобы быстро отчитаться.

**Критерии приемки:**
- [ ] Чек формируется автоматически после оплаты
- [ ] Доступны форматы PDF и Excel
- [ ] Отправка на email происходит автоматически
- [ ] В чеке указаны: дата, маршрут, сумма, реквизиты

## 📊 Метрики успеха

- 100% завершенных поездок имеют чек
- Время доставки чека на email ≤ 1 минута
- Процент бизнес-клиентов, использующих автовыгрузку: ≥ 90%
