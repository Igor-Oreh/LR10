# US-01: Регистрация и вход в систему

## 📜 Формулировка требования

Как пассажир, я хочу зарегистрироваться или войти в приложение, чтобы получить доступ к функциям заказа такси.

## 👥 Роли пользователей

- Неаутентифицированный пользователь (новый/существующий)
- Студент (предпочитает вход через соцсети)
- Бизнес-клиент (нуждается в привязке корпоративного аккаунта)
- Пенсионер 60+ (предпочитает вход по номеру телефона)

## 📋 Детализация по персонам

### Студент Игорь (20 лет)
**User Story:** Как студент, я хочу войти через Telegram/Google, чтобы не тратить время на СМС.

**Критерии приемки:**
- [ ] Кнопки "Войти через Google" и "Войти через Telegram" отображаются на экране входа
- [ ] После успешной авторизации через соцсеть создается/находится аккаунт
- [ ] Переход на главный экран происходит автоматически

### Бизнес-аналитик Игорь (38 лет)
**User Story:** Как сотрудник, я хочу привязать корпоративный аккаунт при входе, чтобы использовать корпоративные функции.

**Критерии приемки:**
- [ ] После входа доступна опция "Привязать корпоративный аккаунт"
- [ ] При вводе корпоративного email система проверяет аккаунт
- [ ] После привязки активируются бизнес-функции (отчеты, корпоративная оплата)

### Пенсионер Игорь Александрович (67 лет)
**User Story:** Как пенсионер, я хочу входить по номеру телефона с крупными цифрами, чтобы не ошибиться при вводе.

**Критерии приемки:**
- [ ] Поле ввода номера телефона имеет увеличенный размер шрифта (минимум 18sp)
- [ ] Клавиатура отображается с крупными цифрами
- [ ] SMS с кодом приходит в течение 30 секунд
- [ ] Поле ввода кода также имеет крупный шрифт

## 📊 Метрики успеха

- Конверсия из открытия приложения в успешный вход ≥ 85%
- Время входа (от открытия до главного экрана) ≤ 60 секунд
- Процент успешных входов с первого раза ≥ 95%
