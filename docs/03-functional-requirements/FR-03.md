# FR-03: Создание и обновление профиля пользователя

## 📜 Формулировка требования

Система должна создавать новый профиль пользователя при регистрации и позволять обновлять настройки профиля.

## 🏷 Атрибуты

| Параметр | Значение |
|----------|----------|
| Источник | Use Case UC-01 (шаги 5-6), US-02 |
| Приоритет | Must Have |
| Стабильность | Стабильно |
| Сложность | Средняя |

## 📋 Детальная спецификация

### Входные данные
- Номер телефона (обязательно), Имя (опционально), Язык, Валюта, Режим интерфейса

### Бизнес-правила
1. При первом входе профиль создается автоматически
2. Для пользователей 60+ автоматически включается крупный интерфейс
3. Настройки сохраняются между сессиями

### Выходные данные
- ID профиля, Сохраненные настройки, Время создания/обновления

## 🔗 Связанные требования

**Реализует:** [US-01](../02-user-requirements/US-01.md), [US-02](../02-user-requirements/US-02.md)
**Зависит от:** [NFR-06](../04-non-functional-requirements/NFR-06.md)

## ✅ Критерии приемки

- [ ] Профиль создается при первой регистрации
- [ ] Настройки сохраняются и применяются
- [ ] Для 60+ автоматически включается крупный интерфейс
