# FR-02: Проверка кода подтверждения

## Формулировка требования

Система должна проверять введенный пользователем код подтверждения и создавать/находить аккаунт.

## Атрибуты

| Параметр | Значение |
|----------|----------|
| Источник | Use Case UC-01 (шаги 3-4) |
| Приоритет | Must Have |
| Стабильность | Стабильно |
| Сложность | Низкая |

## Детальная спецификация

### Входные данные
- Номер телефона, Введенный код (6 цифр)

### Бизнес-правила
1. Код сверяется с отправленным
2. Проверяется срок действия (5 минут)
3. Считаются попытки ввода (макс. 3)
4. При успехе: создается новый аккаунт или находится существующий

### Выходные данные
- Результат проверки, Токен аутентификации, ID пользователя

### Обработка ошибок
- E1: Неверный код → "Неверный код, осталось X попыток"
- E2: Код истек → предложить отправить новый
- E3: Превышен лимит попыток → блокировка на 15 минут

## Критерии приемки

- [ ] Верный код принимается
- [ ] Неверный код отклоняется с подсчетом попыток
- [ ] Истекший код требует повторной отправки
- [ ] При успехе создается сессия
